# 景区搜索与推荐指南

本文档定义景区搜索策略、信息收集维度、展示格式和特殊提醒规则。执行第 2 步（联网搜索）和第 3 步（景点推荐）前阅读。

## 搜索策略

### 搜索关键词

按维度分别搜索，每个维度至少用 2 组关键词：

```
1. 基础信息："{城市} 必去景点推荐 评分"、"{城市} 热门景点 排行"
2. 门票与预约："{景点名} 门票 价格 预约"、"{景点名} 怎么预约 购票"
3. 开放时间："{景点名} 开放时间 营业时间"、"{景点名} 淡旺季 开放时间"
4. 体力与路线："{景点名} 游览路线 推荐"、"{景点名} 游玩攻略 步行 爬山"
5. 最佳时段："{景点名} 最佳游览时间 什么时候去最好"、"{景点名} 上午去还是下午去"
6. 客流高峰："{景点名} 人多吗 排队"、"{景点名} 避开人流 攻略"
7. 景区内交通："{景点名} 电瓶车 缆车 游船"
8. 交通到达："{景点名} 怎么去 交通攻略"、"{景点名} 地铁 公交 直通车"、"{城市} 旅游专线 景区直通车"
```

### 信息来源与可信度

**高可信（优先采信）**：
- **景区官网/官方公众号**：开放时间、门票价格、预约规则最权威
- **携程/美团/大众点评门票页**：实时票价、用户评价
- **高德/百度地图**：位置、距离、交通方式

**中可信（参考但需验证）**：
- **马蜂窝/知乎攻略**：游览路线、体力评估、最佳时段等体验类信息较有价值
- **小红书游记**：照片和体验描述丰富，但需警惕软文
- **大众点评评分**：整体可靠但关注评分分布而非单一分数

**低可信（不单独作为推荐依据）**：
- **抖音/快手探店视频**：付费推广比例高
- **旅行社/OTA 打包推荐**：可能有商业利益

### 推荐指数评分标准

以大众点评/携程评分为主，换算为星级：

| 平台评分 | 星级 |
|----------|------|
| 4.5 及以上 | ★★★★★ |
| 4.0-4.4 | ★★★★☆ |
| 3.5-3.9 | ★★★☆☆ |
| 3.0-3.4 | ★★☆☆☆ |
| 3.0 以下 | ★☆☆☆☆ |

**重要规则**：
- 不同平台的分数**不互相换算**（大众点评 4.3 和携程 4.7 不可直接比较）
- 如果一个景点只有马蜂窝/小红书评分，标注"网友评分 X.X"而非星级
- 优先采用大众点评或携程的评分作为星级依据
- 同一景点多个平台评分差异大时，取大众点评/携程的分数，并在推荐理由中说明

## 信息收集维度

每个景点收集以下信息，分为**必收**和**应收**两级：

### 必收信息（每个景点必须有）

| 字段 | 说明 | 示例 |
|------|------|------|
| 名称 | 景区全称 | 成都大熊猫繁育研究基地 |
| 推荐指数 | 1-5 星 | ★★★★★ |
| 门票价格 | 含优惠信息 | ¥55（学生半价 ¥28） |
| 建议游玩时长 | 区间值 | 3-4 小时 |
| 推荐理由 | 1-2 句核心卖点 | 近距离观看大熊猫，早上去最活跃 |

> 门票优惠金额根据 `ticket-discounts.md` 中的政策和用户优惠身份计算，不要凭估算。

### 应收信息（能搜到就收集）

| 字段 | 说明 | 示例 |
|------|------|------|
| 是否需要预约 | 是/否 + 预约渠道 + 提前天数 | 需预约，微信公众号提前 1 天 |
| 开放时间 | 含末班入园时间 | 7:30-18:00（末班入园 17:00） |
| 体力需求 | 地形 + 步行距离 + 代步工具 | 全程平路，可坐电瓶车 ¥20 |
| 最佳游览时段 | 一天中 + 季节 | 早上 7:30-9:00 最佳，秋季最美 |
| 客流高峰 | 高峰时段 + 避开建议 | 上午 9-11 点最挤，建议 7:30 入园 |
| 游览路线 | 推荐路线 + 必看亮点 | 南门进→月亮产房→太阳产房→西门出 |
| 景区内交通 | 类型 + 费用 + 是否必要 | 电瓶车 ¥20（景区大，建议坐） |
| 到达方式 | 所有可行交通方式（地铁/公交/直通大巴/打车/自驾等） | 地铁 3 号线 + 景区直通车 / 打车 ¥60 / 停车场 ¥10 |
| 配套设施 | 卫生间/餐饮/寄存 | 景区内有餐厅，有行李寄存 |
| 适合人群 | 是否适合老人/小孩/轮椅 | 适合全家，有电瓶车可坐 |
| 网友真实评价 | 从搜索结果中摘取 1-2 条有代表性的真实用户评价 | "带娃去的，早上 8 点入园人少，小熊猫超可爱" |

## 展示格式

推荐列表按推荐指数排序，每个景点展示格式：

```
{推荐指数} {景点名称}
  门票: {价格} | 游玩: {时长}
  🚗 到达: {最推荐的方式} | {备选方式}（如多种则列出）
  🕐 开放: {时间}（末班入园 {时间}）（如应收信息中有则显示此行）
  ⏰ 需预约：{预约方式和提前天数}（如需要则显示此行）
  🏃 体力: {体力描述}（如有特殊需求则显示此行）
  📍 最佳时段: {一天中/季节}（如有明显偏好则显示此行）
  推荐理由: {2-3 句话，包含景点特色、核心亮点、为什么值得去}
  💬 网友说: "{从搜索结果中摘取的真实用户评价}"（如搜到则显示此行）
```

**推荐理由要求**：不写套话（如"值得一去"），要具体描述景点的独特之处。结合搜索结果说明这个景点有什么特别的、适合什么人群、什么体验最突出。

**网友真实反馈**：从 WebSearch 搜到的大众点评、携程、小红书等平台的真实用户评论中摘取 1-2 条有代表性的评价。要求：
- 必须是搜索结果中的真实评论，不能编造
- 选择有信息量的评价（如游玩建议、体验细节），不要纯夸"好看""推荐"
- 如果搜不到有价值的真实评论，此行省略，不伪造

完整示例：

```
成都景点推荐（已搜索到 15 个景点，按推荐指数排序，推 10 个供你选）：
```

## 路线优化原则

景点推荐完成后、进入行程草案前，按以下原则排列每日景点顺序：

1. **地理就近**：同一天的景点优先选择地理位置相近的，避免在城市两端来回跑。搜索各景点的位置信息，按区域分组安排
2. **交通最优**：每两个相邻景点之间选择**耗时最短或费用最低**的交通方式，多个方案都列出来，推荐一个最优的
3. **顺序合理**：考虑一天的游览顺序是否合理（如先远后近还是先近后远），避免走回头路
4. **餐饮就近**：午餐/晚餐尽量安排在景点附近或顺路的餐厅，不为了吃饭跨城区跑

这些优化在行程草案（第 5 步）中体现，每个景点间的交通都必须标注。

## 景点间交通标注规则

行程草案中，每两个相邻活动（景点→景点、景点→餐厅、餐厅→景点等）之间**必须标注交通方式**，至少列出 2 种方案。

### 搜索关键词

```
1. "{景点A} 到 {景点B} 怎么去"
2. "{景点A} 到 {景点B} 公交 地铁"
3. "{景点A} 到 {景点B} 打车 多久 多少钱"
4. "{景点A} 到 {景点B} 步行 多远"
5. "{城市} 景点之间 交通攻略"
```

如果两个景点间搜不到直达交通，根据距离估算：步行（< 2km）、骑行（2-5km）、公交/地铁（5-15km）、打车（任意距离）。

### 标注格式

在行程草案中，景点间交通用以下格式展示：

```
🚲 → 🚶 步行 约15分钟 免费 ⭐ | 🚕 打车 约5分钟 ¥10
   推荐步行，沿途有老城夜景
```

### 标注规则

1. **至少 2 种方案**：必须列出至少 2 种可行的交通方式
2. **推荐标注**：用 ⭐ 标记推荐方案，只能推荐 1 个
3. **推荐理由**：简短说明为什么推荐（≤30字），如"步行沿途风景好""打车最快""公交最省钱"
4. **费用标注**：打车需标注预估费用区间，公交/地铁标注票价，步行/骑行标"免费"
5. **耗时标注**：每种方案都要标注预估耗时
6. **特殊提醒**：偏远景区标注末班车时间；无公共交通的标注"建议包车/打车"
7. **JSON 填写**：生成 JSON 时，交通信息填入每个活动的 `transit` 字段（格式见 `docx-layout.md`）

## 预约提醒规则

**必须标注"需预约"的场景**：
- 景区官方明确要求预约（如故宫、国家博物馆等）
- 搜索结果中出现"限量""限流""需提前预约"等关键词
- 节假日/旺季热门景区（即使平时不需要预约，旺季也可能限流）

**预约信息必须包含**：
- 预约渠道（微信公众号/官网/携程/美团等）
- 提前天数（提前 1 天 / 3 天 / 7 天）
- 是否需要身份证原件
- 放票时间（如"每天 20:00 放第二天的票"）

## 体力需求标注规则

根据景点地形，用以下分级标注：

| 分级 | 标注内容 | 示例 | 代步工具降级 |
|------|----------|------|--------------|
| 轻松 | 全程平路或有电瓶车 | "全程平路，轻松" | — |
| 中等 | 有台阶但不多，步行 1-2 小时 | "有少量台阶，步行约 1.5 小时" | — |
| 较好 | 需要较多步行或爬山 30-60 分钟 | "爬山约 40 分钟，台阶较多" | 有缆车/索道可降为中等或轻松 |
| 暴走 | 全程爬山或长距离徒步 2 小时以上 | "全程爬山 4-6 小时，体力要求高" | 有缆车/索道可降为较好，有全程电瓶车可降为中等 |

**标注规则**：当景点有代步工具且能显著降低体力需求时，必须同时标注原始等级和降级后等级，如：

```
🏃 体力: 较好（全程爬山约 40 分钟）→ 坐索道可降为轻松（索道 ¥65，5 分钟到顶）
```

这样用户看到后可以根据自己是否愿意坐索道来判断，不会因为看到"较好"就直接放弃。

**与用户体力状况的匹配**（考虑代步工具降级后的等级）：
- 用户体力"轻松" → 推轻松级 + 降级后可达轻松级的景点（标注"坐索道/电瓶车即可轻松游览"）
- 用户体力"中等" → 推轻松和中等 + 降级后可达中等级的景点
- 用户体力"较好" → 推轻松到较好 + 降级后可达较好级的景点
- 用户体力"暴走" → 全部可推

**景区内代步工具**必须标注：
- 类型（电瓶车/缆车/索道/游船/滑道）
- 费用
- 是否建议乘坐（如"景区大，建议坐电瓶车"或"步行即可，不必坐车"）

## 最佳游览时段标注规则

**一天中的最佳时段**（有明显偏好时才标注）：
- 早去型：如熊猫基地（早上活跃）、热门景区（避开人流）
- 下午型：如博物馆（避开上午高峰团）
- 傍晚型：如古镇/古街（灯光氛围好）、日落观景点
- 清晨型：如日出观景点（需凌晨出发）

**季节影响**（出发月份与最佳季节不符时标注）：
- 枯水期提醒（瀑布、湖泊类景区）
- 花期/红叶期提醒（自然景观类）
- 封山/封路提醒（高海拔景区冬季）
- 雨季影响（山区景区）

## 游览路线标注规则

大景区（游玩时长 > 2 小时）标注推荐路线：
- 用箭头表示：`南门进 → 月亮产房 → 太阳产房 → 西门出`
- 标注必看亮点
- 标注全程大概步行距离或时间
- 如有多条路线，推荐最适合用户体力的一条

## 景区到达方式

每个景点标注从市中心或用户住宿区域如何到达，**按便利程度列出所有可行方式**：

| 方式 | 标注内容 | 示例 |
|------|----------|------|
| 地铁/轻轨 | 线路 + 站点 + 出站步行距离 | 地铁 3 号线熊猫大道站，D 口出步行 1.5km 或转接驳车 |
| 公交 | 线路号 + 站点 + 大约时间 | 公交 198 路直达景区大门，约 50 分钟 |
| 景区直通大巴/旅游专线 | 上车点 + 班次间隔 + 票价 | 武侯祠有直通大巴至都江堰，每小时一班 ¥30 |
| 打车/网约车 | 大约时间 + 预估费用 | 打车约 40 分钟 ¥60-80 |
| 景区接驳车/摆渡车 | 上车点 + 费用 + 是否免费 | 游客中心有免费摆渡车，每 15 分钟一班 |
| 自驾 | 停车位置 + 停车费 | 景区有大型停车场 ¥10/次，旺季建议早到 |
| 共享单车/步行 | 仅适用于近距离景点 | 从酒店骑车约 15 分钟 |

**展示原则**：
- 有地铁/公交直达的**优先推荐**公共交通，标注线路和站点
- 有直通大巴/旅游专线的**必须标注**，很多游客不知道这种方便又便宜的方式
- **偏远景区如有公共交通，必须标注末班车时间**（如"末班 17:00"），错过就回不来
- 没有末班信息的提醒用户"注意查末班时间，错过只能打车约 ¥XX"
- 多种方式并存时，按性价比排序展示（公共交通 > 直通大巴 > 打车）
- 偏远景区无公共交通时，标注打车费用或建议拼车/包车

## 与出行身份的匹配

根据用户在第 1 步选择的出行类型，在推荐时额外提示：

| 出行类型 | 额外提示 |
|----------|----------|
| 有老人 | 标注景点是否有轮椅通道、休息区多不多、卫生间距离 |
| 有小孩 | 标注是否有儿童项目、是否需要抱/推车、卫生间有无母婴室 |
| 情侣 | 标注适合拍照的景点、浪漫氛围好的时段 |
| 朋友团 | 标注互动性强的项目（漂流、攀岩、密室等） |

## 搜索失败恢复

- 景点基础信息搜不到 → 换关键词重试：去掉"评分"等限定词
- 预约/开放时间搜不到 → 标注"建议出发前查官方公众号确认最新信息"
- 体力需求搜不到 → 根据景区类型和面积估算（山区 > 古城 > 博物馆 > 公园）
- 景区内交通搜不到 → 不标注，用户到了现场看指示牌
